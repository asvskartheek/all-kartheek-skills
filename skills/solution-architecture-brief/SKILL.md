# Solution Architecture Brief

Use this skill when the input is requirements-heavy and the desired output is an architecture proposal or summary.

## Workflow

1. Read the source docs completely.
2. Extract goals, users, constraints, integrations, and deployment requirements.
3. Identify assumptions vs explicit requirements.
4. Produce a clear architecture with named components and boundaries.

## Recommended output

### Problem / goals
### Constraints / non-functional requirements
### Proposed components
### Data flow
### Deployment topology
### Security / compliance / observability notes
### Open questions / assumptions

## Good use cases

- on-prem or cloud architecture briefs
- AI-agent system blueprints
- architecture summaries from markdown-heavy repos
